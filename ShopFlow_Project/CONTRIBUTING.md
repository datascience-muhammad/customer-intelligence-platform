# Contribution Guidelines

## Branching
- Use `feature/<task-name>` for all work.
- Target your team branch for PRs.

## Commit Messages
Format:
<type>: <short description>
Examples:
- feat: added customer ETL pipeline
- fix: corrected null handling in model

## Pull Requests
- Assign reviewer from your sub-team.
- Include summary of changes.
- Ensure no notebook outputs or large files are committed.

## Reviews
- At least 1 approval required.
- Address comments before merging.
